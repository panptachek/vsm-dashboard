### A4. Карта — собрать все баги в один починочный спринт

По скриншоту (см. вложения чата) видно:
- **Нет усов на концах мостов** — должны быть два уса 45° наружу на каждом конце пары параллельных линий.
- **Свайные поля при приближении рисуются коряво** — синие штрихи вместо прямоугольника-вдоль-оси с двумя внутренними ⊥-линиями.
- **Выносы = просто буквы N** — должно быть **пунктирная линия ⊥ оси + N между штрихами**.
- **Труб и путепроводов не видно** — либо рендер отсутствует, либо они микроскопические при данном zoom.
- **Километровые значки справа/сверху над осью** — должны быть **снизу**, на **leader-палочке длиной 2 × диаметр значка**, перпендикулярно оси в данной точке.

Чинить строго по `Условные_обозначения_карты.html` (в uploads — это источник истины, прочитай перед кодом).

Ключевой фикс по зуму:
- Объекты маленькие на zoom < 14 (чтобы не загораживали друг друга, когда их много на экране).
- Растут с зумом: от zoom 14 — нормальный размер, от zoom 16 — крупнее.
- Формула: `sizePx = clamp(baseSize * (zoom - 12) / 4, minSize, maxSize)`.
- Для труб/путепроводов: на zoom < 13 прячутся (они точечные, в кучке сливаются); на zoom ≥ 13 появляются как мелкие значки ⊥ оси, на zoom ≥ 15 — полноразмерные с усами.

Км-значки перенести вниз от оси:
1. В точке ПК посчитать `tangent = getTangentAngleAtPicketage(km * 100)`.
2. Нормаль к оси = `tangent + 90°`. «Вниз» на экране = выбирай знак так, чтобы y (в screen coords) был больше.
3. Сдвиг вдоль нормали = `2 × diameter_значка`.
4. На уровне SVG: leader-линия пунктирная `#999`, 1px, соединяет точку оси со значком.
5. Сам значок не поворачиваем (всегда читаемый).

Попапы:
- Клик по любому типу объекта открывает попап. Сейчас путепроводы молчат — починить `pointer-events` и z-index.
- Невидимая hit-area вокруг каждой иконки (прозрачный path шире визуальной линии), чтобы кликать удобно было.

**DoD:** скрины при трёх уровнях зума (12, 14, 16). На всех — иконки видны пропорционально, км-значки снизу на палочках, попап по клику на каждый из 6 типов.

> Процент выполнения нормы.

> **Формула:** Фактическая выработка за сутки ÷ (Фактическое кол-во машин × Норматив × 2 смены)
>
> **Норматив:** 16 м³ за рейс × плановое кол-во рейсов в смену
>
> **Пример:** 2 620 м³ ÷ (13 машин × 16 × 6 × 2) = 10.5%

### D1. Где лежат дизайнерские файлы

```
src/

│   ├── overview/
│   ├── analytics/
│   ├── map/
│   ├── daily-quarry-report/
│   └── ...
└── pages-wip/                ← дизайнерская пачка (новые WIP-страницы)
    ├── core.jsx
    ├── shell.jsx
    ├── overview.jsx
    ├── analytics.jsx
    ├── daily_roads.jsx
    ├── map_drawer.jsx
    ├── secondary.jsx
    └── app.jsx

styles/
├── globals.css
└── tokens-wip.css            ← дизайнерский токенов, отдельно

data/
└── mock.js                   ← тоже положить, но НЕ использовать в продакшн-роутах
```

### D2. Перевод из standalone HTML в прод-бандл

Дизайнер выдал код под Babel-standalone + CDN React (прямо в браузере), у нас в прод-бандле Vite + React. Надо **адаптировать**, не **копировать как есть**:

1. Конвертировать `.jsx` под Vite (убрать `type="text/babel"`, добавить правильные import'ы).
2. Recharts / Leaflet — уже стоят в `package.json`, использовать модульные импорты.
3. Шрифты Oswald/Saira/JetBrains Mono — подключить через `index.html <head>` или `@import` в CSS, как удобнее.
4. `data/mock.js` — пока оставить как fallback, но в WIP-страницах сразу сделай заглушки `useEffect` + `fetch('/api/...')` с данными из БД, а mock — только если API не ответил.

### D3. Роутинг WIP-вкладок

Каждая WIP-страница получает роут с префиксом `/wip/*`:

```
/wip/overview          →  Обзор (WIP)
/wip/analytics         →  Аналитика (WIP)
/wip/daily-roads       →  Отсыпка АД (WIP)
/wip/map               →  Карта (WIP)
/wip/secondary         →  (что там в secondary.jsx — смотри)
```

В сайдбаре — отдельная **секция** «WIP (дизайн)», под ней WIP-пункты с суффиксом **«(WIP)»** в подписи. Старые пункты остаются как были.

Пример сайдбара после:

```
ОСНОВНОЕ
├── Обзор
├── Карта
├── Аналитика
├── Отчёт по выработке автосамосвалов

WIP (дизайн)
├── Обзор (WIP)
├── Карта (WIP)
├── Аналитика (WIP)
├── Отсыпка АД (WIP)
└── …
```

### D4. Данные в WIP-страницах — из той же БД
